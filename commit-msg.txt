perf: scale the collective brain server-side to 500k facts

The website rebuilt the entire HDC brain in every visitor's browser from
all facts, which allocated multiple GB at ~150k facts and made the page
hang. Recall now runs against a memory-efficient server-side brain and the
browser only loads small counters, a visualisation sample, and query
results.

@hiperbrain/core: make KnowledgeBrain scale to hundreds of thousands of
facts. Cache symbols only bit-packed (~1.25 KB vs ~10 KB), rebuild
per-subject records lazily instead of keeping a dense 40 KB accumulator
each, do binding/majority in the packed domain, and generate symbols
32 bits per draw. A 150k-fact brain drops from multiple GB to ~550 MB and
builds in seconds; similarConcepts is an order of magnitude faster. Recall
results are unchanged (HDC is basis-independent).

Website:
- add un-metered server endpoints: /api/brain/{query,stats,sample,thought,
  benchmark} and /api/facts (paginated, searchable)
- useCollectiveBrain no longer builds a client brain; it loads stats + a
  bounded sample and queries the server
- command bar, home hero, activity log and benchmark now recall server-side
- raise MAX_FACTS to 500,000; keep the server brain warm (15 min TTL,
  rebuilt on fact-count change)

Docs: correct claims that recall runs "in your browser" - the collective
brain now runs server-side; the @hiperbrain/core SDK still runs locally.
